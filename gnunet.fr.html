Introduction

GNUnet est un framework pour reseautage pair a pair securitaire. La premiere application implementee dans GNUnet est un systeme de partage de fichier anonyme et resisant a la censure

Le but du projet GNUnet est de fournir une infrastructure pour un réseau pair à pair sécurisé. Toutes les communications dans GNUnet sont authentifiées et cryptées de lien en lien. Le modèle économique rend les attaques du réseau plus difficiles car ce système économique peut être utilisé pour contrôler l’usage des ressources. Les pairs sur GNUnet echangent des messages grace a un module remplacable abstrayant le service de transport. Actuellement, des services de transport basés sur UDP, TCP, HTTP et SMTP sont disponibles. Le noyau GNUnet fournit des mécanismes pour réaliser les allocations de ressources en CPU, bande passante et espace de stockage. Le noyau garantit le respect des limitations de ressources fixées par l’utilisateur. GNUnet ne repose sur aucun service centralisé. De nouveaux protocoles pair à pair peuvent être facilement implémentés par dessus l’infrastructure de base de GNUnet. Les protocoles actuels sont le partage de fichiers anonyme, un protocole de discussion rudimentaire, des tests de rapidité du débit des messages et la visualisation de la topologie du réseau.
Alors que nos objectifs sont similaires à ceux de projets comme Freenet, Gnutella, Mnet ou d’autres, nous espérons apporter une combinaison supérieure de fonctionnalités pour les utilisateurs qui privilégient la sécurité par rapport à la performance.

Une liste de questions fréquemment posées est disponible. Quelques articles peuvent apporter un aperçu supplémentaire. La documentation exhaustive peut être trouvée sur gnunet.org.

Téléchargement de GNUnet

GNUnet peut être trouvé sur ftp://ftp.gnu.org/gnu/gnunet/, sur les miroirs GNU ou sur gnunet.org.

Outils nécessaires
  • GNU/Linux, FreeBSD, OpenBSD, NetBSD, OS X, Windows XP or Solaris.
  • GNU MP Bignum Library (>= 4.0.0)
  • libgcrypt (>= 1.2.0)
  • gtk+ (>= 2.6)
  • libextractor (>= 0.5.4)
Veuillez garder à l’esprit que ce n’est qu’une version bêta. Il peut encore y avoir des problèmes de stabilité, d’adaptabilité et de performances.
Le réseau est encore très petit. Très peu de contenu est disponible pour le moment. Une inscription aux listes de diffusion de GNUnet peut être une bonne idée.
